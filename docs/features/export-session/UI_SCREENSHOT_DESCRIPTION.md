## Interaction Flow

### 1. Initial State
- User is on Generated Posts page
- They see a table of generated posts
- Each row has "Edit" and "View Session" buttons

### 2. Click "View Session"
- Modal opens with session details
- Header shows "View Session" title
- NEW: "Copy Session JSON" button appears in header (blue, next to close button)
- Modal displays logs and AI call tabs

### 3. Click "Copy Session JSON"
- Button text changes to "Loading..."
- Button is disabled during fetch
- AJAX request sent to server
- JSON is fetched and processed

### 4. Success
- JSON copied to clipboard automatically
- Button text returns to "Copy Session JSON"
- Button is re-enabled
- Green success notification appears at top of modal body
- Notification says: "Session JSON copied to clipboard!"
- After 3 seconds, notification fades out

### 5. Error (if occurs)
- Button text returns to "Copy Session JSON"
- Button is re-enabled
- Red error notification appears
- Notification shows error message
- After 3 seconds, notification fades out

## Browser Compatibility

### Clipboard API
- **Modern Browsers** (Chrome 63+, Firefox 53+, Edge 79+, Safari 13.1+):
  - Uses `navigator.clipboard.writeText()`
  - Requires HTTPS (except localhost)
  
- **Older Browsers** (IE 11, older Chrome/Firefox):
  - Fallback to `document.execCommand('copy')`
  - Works on HTTP and HTTPS

### CSS Features
- **Flexbox**: Widely supported (IE 11+)
- **CSS Animations**: Supported (IE 10+)
- **Border Radius**: Supported (IE 9+)
