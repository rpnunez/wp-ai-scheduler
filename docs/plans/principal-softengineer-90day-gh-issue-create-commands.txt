# GitHub CLI issue creation commands for the Principal Software Engineer 90-Day plan
# Usage:
# 1) Ensure you are authenticated: gh auth status
# 2) Run from repo root: bash docs/plans/principal-softengineer-90day-gh-issue-create-commands.txt
# 3) These commands target: rpnunez/wp-ai-scheduler

set -euo pipefail

REPO="rpnunez/wp-ai-scheduler"

# Optional: ensure milestone titles exist before issue creation.
# If milestones already exist, these calls are harmless failures due to duplicate titles.
gh api -X POST "repos/$REPO/milestones" -f title="PSE 90-Day Plan - Phase 1" >/dev/null 2>&1 || true
gh api -X POST "repos/$REPO/milestones" -f title="PSE 90-Day Plan - Phase 2" >/dev/null 2>&1 || true
gh api -X POST "repos/$REPO/milestones" -f title="PSE 90-Day Plan - Phase 3" >/dev/null 2>&1 || true

echo "Creating Issue 1..."
gh issue create --repo "$REPO" \
  --title "Architecture: define canonical generation boundary and legacy adapter strategy" \
  --label "architecture" --label "tech-debt" --label "phase-1" --label "high-priority" \
  --milestone "PSE 90-Day Plan - Phase 1" \
  --body-file - <<'EOF'
## Summary
Define a canonical internal generation boundary and isolate legacy template-based input handling behind a single adapter.

## Problem
Generation logic still contains inline branching for legacy and context-driven flows, increasing complexity, test scope, and change risk.

## Scope
- [ ] Inventory all legacy generation entrypoints.
- [ ] Define canonical internal generation input model.
- [ ] Define legacy adapter contract and ownership.
- [ ] Capture design in an ADR or equivalent architecture decision doc.

## Acceptance Criteria
- [ ] Legacy entrypoints are fully enumerated.
- [ ] Canonical generation input model is documented and approved.
- [ ] Adapter boundary and ownership are explicit.
- [ ] Follow-on implementation issues reference this design.

## Risk Controls
- Do not change runtime behavior in this issue.
- Keep scope limited to architecture decisions and boundaries.

## Out Of Scope
- Implementation of adapter internals.
- Refactoring prompt builder or generator code paths.
EOF
